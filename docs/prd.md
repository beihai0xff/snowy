# Snowy 产品需求文档（PRD）

## 1. 文档信息
- 产品名称：Snowy
- 文档类型：PRD（偏研发落地版）
- 当前版本：v0.1
- 更新时间：2026-04-14
- 适用范围：Web 端首发版本（MVP）

## 2. 产品背景
高中生在日常学习中存在三个典型痛点：
1. **知识定位慢**：知识点分散在课本、考纲、题库、讲义中，检索入口割裂，难以快速找到“最相关且可信”的答案。
2. **理解过程弱**：传统搜索往往只能给结论，缺少按高中认知水平展开的讲解、题型映射与易错点提醒。
3. **物理建模门槛高**：学生能看到公式，但很难把公式、条件、图像和真实物理过程联系起来，尤其缺乏参数调节后的直观反馈。

Snowy 的目标是构建一款面向高中生的 AIGC Web 学习平台，通过统一知识索引与大模型推理能力，提供：
- 面向高中课程的高可信知识检索；
- 面向高中物理的“推导 + 2D 图表 + 参数调节”建模能力；
- 面向高中生物的结构化建模与过程理解能力；
- 面向课内学习与题目训练的闭环辅助。

## 3. 产品目标

### 3.1 业务目标
- 打造一个围绕高中学习场景的智能学习入口。
- 在首发阶段验证“知识检索 + 物理建模 + 生物建模”三类核心能力的用户价值。
- 为后续扩展化学实验模拟、个性化练习推荐等能力打基础。

### 3.2 用户目标
- 学生可以在 1 次提问内拿到可读、可追溯、可继续学习的答案。
- 学生可以对物理题的条件、公式和图像进行联动理解。
- 学生可以对生物过程、结构关系和实验变量建立模型化理解。
- 学生可以通过参数调节快速理解变量变化对结果的影响。

### 3.3 MVP 成功标准
- 学生在知识检索场景下平均 30 秒内获得首屏有效答案。
- 检索结果引用命中率、题目相关推荐点击率达到可验证水平。
- 物理建模场景中，完成一次参数调节并看到图表变化的转化率达到预期。

## 4. 目标用户与角色

### 4.1 核心用户
1. **高中生（主用户）**
   - 高一到高三学生
   - 典型诉求：查知识点、查题型、看推导、理解图像、梳理考点
2. **教师/教研人员（次用户）**
   - 用于验证内容质量与教学适配度
   - 典型诉求：检查解释准确性、查看知识映射逻辑
3. **家长（低频旁观用户）**
   - 关注孩子学习效率与理解效果

### 4.2 用户分层
- 基础型：需要快速查概念、公式、定义
- 提升型：需要知识点与题型映射、易错点讲解
- 冲刺型：需要考纲关联、题目变式、参数化理解

## 5. 产品范围

### 5.1 本期首发范围（MVP）
1. **统一索引知识检索**
   - 数据源统一覆盖：课本与考纲、题库与讲义

   - 支持知识点检索、题目检索、考纲定位、讲义片段引用
   - 输出答案摘要、引用原文、关联题目/知识点
2. **高中物理建模**
   - 支持题干解析与已知条件抽取
   - 支持按高中课程范围进行公式匹配与推导说明
   - 支持 2D 图表展示
   - 支持参数调节并实时刷新结果
3. **高中生物建模**
   - 支持围绕核心章节进行结构化建模与过程解释
   - 支持概念关系图、过程流转图、变量影响分析等结果展示
   - 支持实验类题目中的自变量、因变量、控制变量拆解
   - 支持知识检索结果一键跳转至生物建模页面
4. **基础学习闭环**
   - 支持收藏、历史记录、继续追问

## 6. 核心价值主张
- **可信**：回答基于统一索引数据源并附带引用。
- **好懂**：面向高中生表达，减少“只有结论没有过程”的回答。
- **可验证**：所有关键结论尽可能展示出处、公式或图表。
- **可交互**：通过参数调节与图像变化帮助理解抽象概念。

## 7. 核心使用场景

### 7.1 场景 A：知识检索
用户输入“牛顿第二定律在斜面题中怎么用”，系统返回：
- 结论摘要
- 课本/讲义相关段落引用
- 对应考纲标签
- 常见题型与练习入口

### 7.2 场景 B：题目检索与知识回溯
用户输入一道题或上传题干文本，系统：
- 识别题型
- 提取涉及知识点
- 匹配题库中的同类题
- 返回解析思路和推荐练习

### 7.3 场景 C：物理推导
用户输入“平抛运动最大水平位移怎么推导”，系统：
- 识别章节与模型
- 逐步列出推导过程
- 标记每一步所用定律
- 给出条件变化下的推导差异提示

### 7.4 场景 D：物理可视化与参数调节
用户选择某个物理模型（如匀变速直线运动、平抛、弹簧振子），系统：
- 展示变量输入面板
- 生成 2D 图表（位移-时间、速度-时间、受力关系等）
- 用户调整参数后，图表、关键结果和结论同步刷新

### 7.5 场景 E：生物建模与过程理解
用户输入“光合作用中光照强度变化如何影响有机物积累”或某道实验题，系统：
- 识别涉及章节与核心概念（如光合作用、呼吸作用、限制因素）
- 拆解变量关系与过程阶段
- 生成结构化过程说明、关系图或流程图
- 给出实验设计思路、易错点与关联知识点

## 9. 功能需求明细

## 9.1 模块一：统一索引知识检索

### 9.1.1 功能说明
将课本与考纲、题库与讲义统一进入同一知识索引层，提供自然语言搜索与结构化召回能力。

### 9.1.2 用户故事
- 作为高中生，我希望输入一个自然语言问题，就能得到简洁准确的知识答案。
- 作为高中生，我希望知道答案来自哪里，而不是只看模型生成结果。
- 作为高中生，我希望看到相关题目，马上检验是否真正理解。

### 9.1.3 需求列表
1. 支持自然语言问答
2. 支持按学科、年级、章节、题型过滤
3. 支持返回引用片段与来源标签
4. 支持展示知识点标签、考纲映射
5. 支持关联题目推荐
6. 支持继续追问与上下文延续

### 9.1.4 输入
- 文本问题
- 题干文本
- 可选筛选项

### 9.1.5 输出
- 答案摘要
- 核心知识点
- 原文引用
- 关联题目
- 推荐下一步学习动作

### 9.1.6 验收标准
- 至少返回 1 条有效引用
- 回答需包含知识点标签
- 若命中题库数据，则需至少给出 1 题关联练习
- 用户点击引用后可定位来源元数据

## 9.2 模块二：物理建模（推导 + 2D 图表 + 参数调节）

### 9.2.1 功能说明
围绕高中物理典型模型，提供从题干理解、变量抽取、公式匹配、推导说明到二维图形展示的完整链路。

### 9.2.2 首发覆盖模型
- 匀速/匀变速直线运动
- 平抛运动
- 牛顿第二定律与受力分析
- 功和能基础模型
- 简单双体运动关系

### 9.2.3 用户故事
- 作为高中生，我希望系统不是只给答案，而是按步骤解释为什么这么列式。
- 作为高中生，我希望看到图像随参数变化而改变，从而理解变量关系。
- 作为高中生，我希望把题目条件直接转化为模型输入，减少手动搭建成本。

### 9.2.4 需求列表
1. 支持题干文本解析
2. 支持变量与单位抽取
3. 支持公式匹配与推导链展示
4. 支持关键变量编辑与参数滑杆调节
5. 支持生成 2D 图表
6. 支持显示结论区、注意事项区、适用条件区
7. 支持从知识检索页跳转并自动带入上下文

### 9.2.5 输出结果结构
- 模型识别结果
- 条件清单
- 推导步骤
- 最终表达式/数值结果
- 2D 图表
- 参数说明
- 易错点提示

### 9.2.6 验收标准
- 生成结果中必须包含结构化条件清单
- 至少展示一类二维图表
- 参数修改后 2 秒内完成图表刷新
- 推导步骤应按序编号展示

## 9.3 模块三：生物建模（结构化建模 + 过程关系展示）

### 9.3.1 功能说明
围绕高中生物核心章节，提供从题意理解、概念识别、过程拆解到变量关系建模的完整链路，帮助学生建立对生命过程与实验逻辑的结构化认知。

### 9.3.2 首发覆盖主题
- 光合作用与细胞呼吸
- 遗传的基本规律
- 生态系统中的能量流动与物质循环
- 细胞结构与物质运输
- 生物实验设计基础

### 9.3.3 用户故事
- 作为高中生，我希望系统能把复杂的生物过程拆成可理解的阶段和因果关系。
- 作为高中生，我希望在做实验题时快速识别自变量、因变量和控制变量。
- 作为高中生，我希望看到知识点之间的结构关系，而不是零散记忆。

### 9.3.4 需求列表
1. 支持题干文本与知识点描述输入
2. 支持概念实体识别与关系抽取
3. 支持过程分阶段拆解
4. 支持生成关系图、流程图或变量分析卡片
5. 支持实验设计题的变量识别与规范表达
6. 支持输出相关考纲标签、易错点与关联练习
7. 支持从知识检索页跳转并自动带入上下文

### 9.3.5 输出结果结构
- 主题识别结果
- 核心概念清单
- 过程拆解步骤
- 变量关系说明
- 结构图/流程图
- 实验设计提示
- 易错点提示

### 9.3.6 验收标准
- 输出中必须包含结构化概念清单与关系说明
- 至少展示一类结构图、流程图或变量关系结果
- 若为实验题，需明确自变量、因变量与控制变量
- 结果应包含至少 1 个相关考纲或知识点标签

## 9.4 模块四：学习闭环基础能力
1. 搜索/建模历史记录
2. 收藏与再次打开
3. 继续追问
4. 分享链接（仅页面级，不涉及社交体系）

## 10. 页面与交互流程

### 10.1 知识检索主流程
1. 用户进入首页或知识检索页
2. 输入问题或题目文本
3. 系统执行检索与重排
4. 系统生成答案摘要与引用内容
5. 用户点击引用查看上下文
6. 用户点击关联题进入练习或建模页

### 10.2 物理建模主流程
1. 用户进入物理建模页
2. 输入问题/题目
3. 系统抽取条件并识别模型
4. 系统展示推导过程和初始图表
5. 用户调节参数
6. 系统刷新图表与关键结论
7. 用户收藏或继续追问

### 10.3 生物建模主流程
1. 用户进入生物建模页
2. 输入问题、知识点或实验题目
3. 系统识别主题、概念与关系
4. 系统展示过程拆解、变量分析和结构图/流程图
5. 用户继续追问或切换相关知识点
6. 用户收藏、回看或进入关联练习

### 10.4 异常流程
- 未召回足够引用时：提示“结果可信度不足”，优先展示检索片段，降低生成比重
- 题干过短或条件不足时：提示用户补充条件
- 模型识别失败时：回退到通用知识讲解模式

## 11. 接口边界（面向研发）

### 11.1 前后端边界
前端负责：
- 输入收集与交互状态管理
- 结果展示与参数面板交互
- 2D 图表渲染

后端负责：
- 内容召回与重排
- 大模型编排与路由
- 结构化答案生成
- 物理建模步骤与数值计算
- 审计日志与结果缓存

### 11.2 核心接口草案
1. `POST /api/v1/search/query`
   - 输入：query、filters、session_id
   - 输出：answer、citations、knowledge_tags、related_questions
2. `POST /api/v1/modeling/physics/analyze`
   - 输入：question、context、selected_model
   - 输出：conditions、derivation_steps、chart_spec、result_summary
3. `POST /api/v1/modeling/physics/simulate`
   - 输入：model_id、parameters
   - 输出：chart_spec、computed_values、warnings
4. `POST /api/v1/modeling/biology/analyze`
   - 输入：question、context、selected_topic
   - 输出：concepts、process_steps、relation_graph、result_summary
5. `GET /api/v1/history`
6. `POST /api/v1/favorites`

## 12. 非功能需求

### 12.1 性能
- 知识检索首屏响应时间：P95 ≤ 3 秒
- 参数调节后二次刷新：P95 ≤ 2 秒

### 12.2 可用性
- 关键接口可用性目标：99.9%
- 模型服务异常时应支持回退策略

### 12.3 内容质量
- 答案需尽量基于引用，不允许纯幻觉型结论直接输出
- 面向高中认知层级，不输出明显超纲表达作为默认答案

### 12.4 安全与合规
- 未成年人场景优先，控制不适内容生成
- 记录关键生成日志与审计信息
- 对外部模型调用进行脱敏与权限控制

## 13. 数据需求
- 内容数据：课本、考纲、题库、讲义
- 内容元数据：学科、年级、章节、知识点、题型、来源、更新时间
- 用户行为数据：搜索词、点击引用、收藏、调参行为、停留时长
- 模型日志：路由模型、token 消耗、耗时、回退原因

## 15. 风险与应对
1. **模型幻觉风险**
   - 应对：统一索引 + 引用约束 + 低可信回退
2. **内容版权与授权风险**
   - 应对：首期只接入具备合法授权的数据源，保留来源与使用日志
3. **物理推导不稳定风险**
   - 应对：首期限定模型范围，增加规则校验与模板化步骤
4. **生物关系抽取不准确风险**
   - 应对：限定首发主题范围，结合知识图谱模板与人工抽检机制
5. **响应慢风险**
   - 应对：缓存热点问题、分层召回、模型路由回退
